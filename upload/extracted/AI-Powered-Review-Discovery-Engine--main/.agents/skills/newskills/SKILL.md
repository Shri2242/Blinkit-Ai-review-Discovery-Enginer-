# Git Workflow

- Conventional commit format: type(scope): description
- Check `git status` and `git diff` before committing — confirm only intended files are staged
- Write commit messages describing *why*, not just *what*, when the change isn't self-explanatory
- Before any rebase/force-push: confirm the branch isn't shared, or get explicit confirmation
